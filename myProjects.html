Sales Representative and deployment Mangement System

Oct- 2020 - Present

The sales Representative Management System is a web-based application developed for Pharma market using Angular and Java technologies, The system is designed to help businesses plan, deploy, and manage their sales representatives efficiently.

...

The system provides an intitive user interfaces that allows Admin and Field users to create and manage sales territories, business rules, territory design, alignment, roster management. Field users van view their assigned territories, manage their appointments and leads.

It helps in better planning and improve efficiency and with proper execution it will help give user with increased sales.

The system uses a PostgreSQL Database to store all the data related to planning and roster management, territories and customer data. The data is securely accessed using RESTful APIs build using Java Spring framework.

Features :

  • User authetication and Authorization : Only authorized users can access the system
  • Sales territory management : Field users can view and manage sales territories
  • Customer and Zip management : Manage and do the alignments for appointments
  • Assignment management : View and manage sales reps and products to deal with customers
  • Bitemporal timeline : Changes can be managed across bitemporal timeline.
  • Report : Sales managers can generate Reports to track the changes and performances.
  • Import and Export : The data can be populated on bulk and Sales managers can export the valid data from DB to track the changes.

Technologies Used :

  • Frontend : Angular 13, HTML 5, CSS 3, TypeScript
  • Backend : Java 11, Spring Boot, Spring MVC, Factory Design Pattern
  • Database : PostgreSQL
  • APIs : RESTful APIs
  • Tools : Visual Studio Code, IntelliJ, Bit Bucket, JUnit, Mockito, Jasmine, Postman
  • Frontend : Angular 13, HTML 5, CSS 3, TypeScript
  • It is designed and implemented using RESTful APIs and JAVA Spring framework
  • Also it is protected with unit and integration tests using JUnit and Jasmine for backend and frontend respectively.
